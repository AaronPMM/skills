# 規則 03:型別與運算

整數運算錯誤在嵌入式的代價特別高:感測器數值、定時器 tick、協議欄位全是整數,而 C 的隱式轉換規則違反直覺。

## 3.1 固定寬度型別

- 涉及硬體暫存器、通訊協議、儲存格式的資料,必須使用 `<stdint.h>` 固定寬度型別(`uint8_t`/`int16_t`/`uint32_t`…),不得用 `int`/`long`(寬度隨平台變化)。
- 循環計數器、陣列索引可用 `size_t`/`uint32_t`;注意 `uint8_t` 計數器在 >255 次迴圈時回繞造成死迴圈。

## 3.2 有號/無號混用(重點檢查)

- 比較運算中一側有號一側無號:有號側被隱式轉為無號,`-1 > 0u` 為真。特別檢查 `if (len - offset > limit)` 這類式子——`len < offset` 時無號減法回繞成巨大正數。
- 無號數判斷「是否小於 0」恆為假,通常暗示邏輯錯誤,編譯器可能連警告都不給。
- 遞減迴圈 `for (uint8_t i = n; i >= 0; i--)` 永不終止。

## 3.3 整數提升與縮窄

- 小於 `int` 的型別(`uint8_t`/`uint16_t`)參與運算前被提升為 `int`(有號!):
  - `(uint8_t)a << 24` 提升為 int 後左移可能位移進符號位(UB);應先轉 `uint32_t`。
  - `~(uint8_t)x` 結果是 int,高位全 1,與 `uint8_t` 比較會恆不等——CRC/校驗和代碼常見。
- 縮窄賦值截斷高位:`uint16_t = u32_val;` 需確認值域或顯式遮罩/註明意圖。

## 3.4 溢位與回繞

- 有號整數溢位是**未定義行為**;無號回繞是定義好的但常非本意。
- 乘法先溢位再賦值給寬型別:`uint32_t us = ms * 1000;` 當 `ms` 是 `uint16_t` 時,乘法在 int 中做,大值時 UB/錯誤;定時計算(ms→ticks、頻率換算)必查。
- **定時器回繞**:用 tick 計時的超時判斷必須寫成 `(uint32_t)(now - start) >= timeout`(利用無號回繞),不得寫 `now >= start + timeout`(回繞時失效)。32 位 ms tick 約 49.7 天回繞——長期運行設備必然遇到。
- 平均值/中點計算 `(a + b) / 2` 的加法溢位。

## 3.5 位操作

- 位移量 ≥ 型別寬度或為負是 UB:`1 << 32`、動態位移量必須驗證範圍。
- 設置/清除位的慣用法檢查:`reg |= (1u << n)`、`reg &= ~(1u << n)`——注意 `1` 要寫成 `1u`(n≥31 時 `1 << 31` 是 UB)。
- 提取位段:遮罩與位移的順序、遮罩寬度是否與欄位定義一致。

## 3.6 浮點

- 無 FPU 的 MCU 上浮點是軟體模擬:慢且增加代碼體積,控制迴路/ISR 中出現浮點要質疑,優先定點運算。
- 浮點相等比較 (`==`) 幾乎總是錯的,用誤差範圍比較。
- 浮點累加誤差:長期運行的積分器 (integrator) 用 float 累加會漂移,檢查是否應用 double 或定點。
- `float` 只有約 7 位有效數字:用 float 存放 epoch 秒、微秒計數會損失精度。

## 3.7 除法與求餘

- 除數可能為 0 的路徑(尤其除數來自測量值、配置、通訊資料)。多數 MCU 除以 0 觸發 fault 或返回垃圾值,不會像桌面一樣立刻崩潰暴露。
- 有號整數除法/求餘對負數的截斷方向 (向零取整) 是否符合意圖。
