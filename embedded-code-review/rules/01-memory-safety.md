# 規則 01:記憶體安全

嵌入式系統 RAM 極其有限、通常無 MMU 保護,記憶體錯誤不會產生 segfault,而是靜默損毀鄰近資料,在數天後以無關症狀爆發。此類問題一律從嚴。

## 1.1 緩衝區邊界(BLOCKER 級)

- 任何寫入緩衝區的操作,必須能證明長度受控:檢查 `memcpy`/`memset`/`strcpy`/`sprintf` 的長度來源。
  - 禁用 `strcpy`/`strcat`/`sprintf`/`gets`;要求 `strncpy`(注意其不保證 NUL 結尾)、`snprintf`、或帶長度參數的自製函數。
  - `snprintf` 返回值是「想寫入的長度」而非「實際寫入的長度」,用返回值做後續偏移前必須先與緩衝區大小比較。
- 來自通訊介面(UART/SPI/CAN/BLE/網路)的長度欄位是**不可信輸入**,使用前必須驗證上限。協議解析代碼是緩衝區溢位的重災區。
- 陣列索引來自外部輸入或計算結果時,檢查是否可能越界(包括負值——索引變數應為無號或先驗證)。
- **Off-by-one 逐一驗證邊界**:迴圈條件 `<=` vs `<`(`for (i = 0; i <= N; i++)` 對大小為 N 的陣列越界一格);字串緩衝區是否給 NUL 結尾留了一位(`strlen(s)` 不含 NUL,拷貝需要 `strlen(s) + 1`);邊界判斷 `idx > MAX` vs `idx >= MAX` 與陣列實際大小的關係。審核時對每個邊界比較代入「恰好等於邊界值」驗算一次。
- `sizeof(ptr)` vs `sizeof(array)`:指標參數上使用 `sizeof` 得到的是指標大小,是常見錯誤。
- 環形緩衝區 (ring buffer):檢查 head/tail 回繞邏輯、滿/空判斷(容量為 2^n 時的遮罩是否正確)、讀寫索引更新順序。

## 1.2 堆疊 (Stack)

- **禁止大型區域變數**:區域陣列 > 256 位元組(或專案規定值)應改為 static 或全域。嵌入式任務堆疊通常只有 512B~4KB。
- **禁止遞迴**,除非能證明深度有嚴格上界且已計入堆疊預算。
- 禁止 VLA(變長陣列)和 `alloca`——堆疊用量不可靜態分析。
- 返回區域變數的位址/引用是 BLOCKER。
- ISR 內的堆疊使用要特別小心:ISR 借用被中斷上下文的堆疊(或專用中斷堆疊),大區域變數會放大所有任務的堆疊預算。

## 1.3 動態記憶體

- 預設立場:**初始化完成後不得使用 `malloc`/`free`**(長期運行的堆碎片化不可預測)。若專案允許動態分配,檢查:
  - 每個 `malloc` 返回值都檢查了 NULL,且失敗路徑合理(不是靜默忽略)。
  - 每條路徑(包括錯誤提前返回)都有對應的 `free`,無洩漏、無 double-free。
  - `free` 後指標置 NULL 或不再可達。
- ISR 中呼叫 `malloc`/`free` 是 BLOCKER(不可重入、可能阻塞)。
- 使用記憶體池 (memory pool) 時:檢查歸還後是否繼續使用 (use-after-free)、池耗盡的處理路徑。

## 1.4 指標

- 指標解引用前的 NULL 檢查:對外部傳入(公開 API 參數、回呼函數參數)必查;內部靜態函數若呼叫端可證明非 NULL 可豁免,但要確認該證明成立。
- 指標運算不得越過所屬物件範圍(比較兩個不相關物件的指標是未定義行為)。
- 函數指標:呼叫前確認已初始化;回呼註冊/反註冊與呼叫之間是否有競態(見規則 02)。
- 型別雙關 (type punning):透過指標強制轉型讀寫不同型別違反 strict aliasing,應使用 `memcpy` 或 union(C 中 union 雙關合法,C++ 中不合法——用 `memcpy` 或 C++20 `bit_cast`)。

## 1.5 初始化

- 所有變數在讀取前必須被初始化。特別注意:錯誤路徑上提前使用了只在成功路徑初始化的變數。
- `.bss`/`.data` 依賴:若代碼假設全域變數為 0,確認啟動代碼確實清零(通常成立,但自訂連結腳本的 no-init 段除外)。
- 結構體部分初始化:`= {0}` 只保證數值成員為零,padding 內容未定義——結構體要 `memcmp` 比較或透過通訊發送前,應 `memset` 整個結構體。
